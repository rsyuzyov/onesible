Порядок разработки:

Переименовать обработку по шаблону "МенеджерРоли[ИмяРоли]"

Реализовать стандартные действия для ролив области "ОбщийПрограммныйИнтерфейс":

ПроверитьРоль(Компьютер)

УстановитьРоль(Компьютер)

УдалитьРоль(Компьютер)

Реализовать прикадные (специфические) действия для роли в области "ПрикладнойПрограммныйИнтерфейс"

В кчаестве примера добавлен метод ВыполнитьНекоеДействие(Компьютер)

Все алгоритмы, как методы, так и "шаги" внутри методов, должны возвращать ответ с результатом выполнения.
В случае проблем должно предоставляться подробное описание ошибки и по возможности лог выполнения операций.

Ответ предоставляется в виде стандартной структуры:

КодВозврата - Число, 0 - ОК, иначе код ошибки (если коды не предусмотрены, то 1)

ОписаниеОшибки - Строка - Объяснение сусти проблемы,когда Кодвозврата <> 0

Лог - Строка - Лог выполнения операции, например стандартный вывод, полученный  при выполнении метода УдаленноеУправление.ВыполнитьCMD
